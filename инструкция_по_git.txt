  Инициализация репозитория с гитхаба
1 установить git и gitflow
2 настроить доступ git к репозиторию на гитхабе https://github.com/neharon/termoduino возможно на шаге 5 будет запрос на доступ если нет надо будет настроить доступ самостоятельно
3 создать папку C:\lang\source\arduino\termo_duino и зайти в неё
4 git init // инициализируем репозиторий в папке
5 git remote add origin  https://github.com/neharon/termoduino //добавляем удаленный репозиторий
6 git fetch // загружаем из удаленного репозитория историю комитов т.е. все изменения которые были внисены другими пользователями и сохранены на гите но только саму историю изменений не изменяя файлы
7 git add . // добавляем все локальные файлы из папки на версионное хранилище в гите
8 git commit -m "initial commit" // делаем первый комит т.е. первый раз фиксируем изменения в локальный репозиторий на компьютере с комментарием из ковычек
9 на всякий случай повторяем пункт 6 
10 git push --set-upstream origin master // загружаем на удаленный репозиторий локальные комиты в нашем случае пока один
11 проверить что не удалился .gitignore, посмотрев в папке и дописать в него требуемые маски файлов

  После инициализации репозитория желательно произвести инициализвцию gitflow. Gitflow это общепринятый способ организации версионного хранилища 
 1 git flow init // дальше все параметры по умолчанию
 2 git checkout develop // переключится на ветку develop


  При необходмости зафиксировать версию:
 1 git add . //если создавались новые файлы команда добавит их в локальный репозиторий
 2 git commit -m "описание изменений" //фиксируем изменения в локальный репозиторий на компьютере с комментарием из ковычек
 3 git fetch // загружаем из удаленного репозитория историю комитов т.е. все изменения которые были внисены другими пользователями и сохранены на гите но только саму историю изменений не изменяя файлы
 4 git pull // загружаю себе последнюю версия изменяя локальные файлы которые были изменены в удаленном репозитории
 5 на этом этапе может потребоваться смержить версии если это не произошло на 4 этапе атоматически но это происходит если разработчики одновременно изменили оди и тотже файл чтобы этого не было перед началом изменений необходимо выполнить пункты 3 и 4
 6 git push // загружаем внесенные изменения с локального на удаленный репозиторий
